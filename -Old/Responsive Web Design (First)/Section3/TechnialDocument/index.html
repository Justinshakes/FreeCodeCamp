Introduction

Java is a cross-platform, object-oriented programming language. It is a robust and versatile language. Inside a host environment, Java can be connected to objects and libraries to provide programmatic control over them.

Java contains a standard library of classes, such as ArrayList, Date, and Math, and a core set of language elements such as operators, control structures, and statements. Core Java can be extended for a variety of purposes by using additional classes and libraries.

What you should already know

This guide assumes you have the following basic background:

  • A general understanding of programming concepts.
  • Good working knowledge of software development principles.
  • Some programming experience. If you are new to programming, consider taking an introductory Java course.
Java and JavaScript

Java and JavaScript are similar in some ways but fundamentally different in others. The Java language is statically typed and has strong type checking, while JavaScript is dynamically typed.

Although JavaScript shares some syntax and naming conventions with Java, it has distinct features such as a prototype-based object model and functions without explicit types.

Java enforces strict rules regarding variable declarations, access modifiers, and interfaces, while JavaScript offers more flexibility.

Hello world
To get started with writing Java, create a class and write your first "Hello world" Java code: public class HelloWorld { public static void main(String[] args) { System.out.println("Hello, World!"); } } Compile and run the code to see "Hello, World!" displayed in your console!
Variables

In Java, you use variables as symbolic names for values in your application. Variable names, known as identifiers, must adhere to certain rules.

A Java identifier must start with a letter, underscore (_), or dollar sign ($), followed by letters, digits (0-9), or underscores. Java is case-sensitive, so "A" and "a" are different identifiers.

You can also use Unicode characters in identifiers, making Java suitable for internationalization.

Declaring variables
You can declare a variable in Java using the following syntax:

With the keyword "int" (for integers): int x = 42; This syntax can be used to declare both local and instance variables.

By simply assigning it a value: x = 42; This always declares a variable and assigns a global variable in the context of a class.

Using other data types and keywords as needed. For example, you can declare a double variable like this: double y = 13.0; This syntax can be used to declare block scope local variables.

Variable scope

In Java, when you declare a variable outside of any method, it is called an instance variable, available to any other code in the current class. When you declare a variable within a method, it is called a local variable, available only within that method.

Java enforces block statement scope, which means a variable declared within a block is local to that block. For example, the following code will work as expected:

if (true) { int x = 5; } System.out.println(x); // This will result in a compilation error

This behavior ensures that variables are more closely scoped and helps prevent unintentional variable reuse.

if (true) { int y = 5; } System.out.println(y); // This will result in a compilation error
Global variables

In Java, global variables are typically declared as static fields within a class. They are accessible to all instances of the class and are associated with the class itself.

Consequently, you can access global variables from other classes by using the class name followed by the field name. For example, if a static variable called "phoneNumber" is declared in a class named "Utility," you can refer to it as "Utility.phoneNumber."

Constants

In Java, you can create a read-only, named constant using the "final" keyword. The syntax for defining a constant identifier is the same as for a variable identifier.

final double PI = 3.14;

A constant cannot change its value after initialization and cannot be re-declared while the program is running.

Constants in Java follow the same scope rules as block scope variables. If you omit the "final" keyword, the identifier is assumed to represent a variable rather than a constant.

You cannot declare a constant with the same name as a function or variable in the same scope.

// This will result in an error: // public void f() {}; // final int f = 5; However, object attributes are not protected, so you can modify them: final MyObject MY_OBJECT = new MyObject(); MY_OBJECT.setValue("otherValue");
Data types

Java defines several data types:

  • Primitive data types:

    • boolean: true or false
    • byte: 8-bit integer
    • short: 16-bit integer
    • int: 32-bit integer
    • long: 64-bit integer
    • float: 32-bit floating-point
    • double: 64-bit floating-point
    • char: 16-bit Unicode character

  • Reference types:

    • Objects: Instances of classes that can store data and have methods to operate on that data.
    • Arrays: Ordered collections of elements of the same data type, accessed by an index.
    • Interfaces: Contracts that define a set of methods that a class must implement.
    • Enums: Special data types that consist of a fixed set of constants.
    • Custom reference types: Classes created by developers to define custom data structures and behaviors.
These data types enable you to perform various operations and store different types of data in your Java applications. Objects and classes are fundamental elements in the language, allowing you to define complex data structures and create reusable code.
if...else statement
Use the "if" statement in Java to execute a block of code if a specified condition is true. You can also use the optional "else" clause to execute a different block of code if the condition is false. The basic syntax of an "if" statement is as follows: if (condition) { // statement_1; } else { // statement_2; } The "condition" can be any expression that evaluates to a boolean value (true or false). If the condition evaluates to true, "statement_1" is executed; otherwise, "statement_2" is executed. Both "statement_1" and "statement_2" can be single statements or compound statements enclosed in curly braces.

You can also use "else if" clauses to test multiple conditions in sequence:

if (condition_1) { // statement_1; } else if (condition_2) { // statement_2; } else if (condition_n) { // statement_n; } else { // statement_last; } In this case, only the first condition that evaluates to true will execute its corresponding block of code.
while statement
A "while" statement in Java executes a block of code as long as a specified condition is true. The basic syntax of a "while" statement is as follows: while (condition) { // statement; } If the "condition" evaluates to true, the "statement" within the loop is executed. After each execution of the "statement," the condition is re-evaluated. If the condition is still true, the loop continues; otherwise, the loop terminates, and control passes to the statement following the "while" block.

You can use block statements enclosed in curly braces to group multiple statements within the loop.

Example:

The following "while" loop iterates as long as "n" is less than three:

int n = 0; int x = 0; while (n < 3) { n++; x += n; }

After each iteration, the loop increments "n" and adds that value to "x." Therefore, "x" and "n" take on the following values:

  • After the first pass: "n" = 1 and "x" = 1
  • After the second pass: "n" = 2 and "x" = 3
  • After the third pass: "n" = 3 and "x" = 6

After completing the third pass, the condition "n < 3" is no longer true, so the loop terminates.

Function declarations
In Java, a function definition (also known as a method declaration) consists of the following parts:
  • The return type of the function, indicating what type of value the function will return.
  • The name of the function, used to identify and call the function.
  • A list of parameters (arguments) enclosed in parentheses, specifying the data types and names of the arguments the function expects.
  • The Java statements that define the function's behavior, enclosed in curly braces, { }.

For example, the following code defines a simple function (method) named "square" in a class:

public int square(int number) { return number * number; }

In this example:

  • The return type is "int," indicating that the function will return an integer value.
  • The name of the function is "square."
  • The function takes one parameter named "number" of type "int."
  • The function consists of one statement that calculates the square of the input number and returns it as an integer.

In Java, primitive parameters (such as "int") are passed to functions by value, meaning that a copy of the value is passed to the function. If the function modifies the parameter's value, it does not affect the original value outside of the function.
